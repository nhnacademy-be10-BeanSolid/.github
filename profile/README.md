## 🔥 CI/CD 파이프라인 구축
<img width="1678" height="488" alt="image" src="https://github.com/user-attachments/assets/80007a0d-1925-497d-afdc-ccfdef496354" />
> 먼저 feature/ 브랜치와 이슈를 만들어 각 인원이 담당한 기능을 개발하고 develop 브랜치로 Pull Request 요청을 보내면 미리 지정된 1~2명의 인원이 함께 code review 를 진행합니다. 그리고 CI 를 통해 빌드 및 테스트를 진행하며 sonarQube 를 통해 정적 코드 분석을 진행하고 Test Coverage 를 저장하는데 실패 시 버그 수정 작업을 거친 후 앞서 진행했던 과정을 다시 시행합니다. 성공했다면 main 브랜치로 병합하고 CD로 서비스에 최종 반영합니다.
